SLIDE 5 — DATA FLOW
====================================================

Title:
Data Flow Pipeline

Step 1: Environment Setup
- File: config_agent.py
- Loads .env, validates 5 API keys, sets up logging

Step 2: Search & URL Collection
- File: search_agent.py
- 6 query categories × 2 search engines = up to ~456 raw URLs
- Queries run in parallel via asyncio.gather
- Normalizes URLs, removes aggregators, deduplicates

Step 3: Async HTML Fetching
- File: scraper_agent.py + pipeline_agent.py
- Batches of 4, shared aiohttp session
- 3 retries, exponential backoff (2s → 4s → 8s)
- robots.txt cached per domain, blacklisted domains skipped

Step 4: LLM Extraction & Classification
- File: scraper_agent.py
- HTML truncated to 15,000 chars
- Single GPT-4o call extracts all fields + classifies
- Throttled by semaphore (concurrency = 2)

Step 5: Post-Processing
- File: pipeline_agent.py
- Keep only "ai_tool" classified results
- Deduplicate by (website, title)
- Remove tools older than 7 days

Step 6: Output
- File: main.py + output/
- Console summary + Teams webhook
- Error report + blacklist saved


====================================================
SLIDE 6 — SEARCH STRATEGY
====================================================

Title:
Search & Filtering Strategy

Dual Search Engines (run in parallel):
- Serper.dev — Google SERP API, async POST
- SerpAPI — Google Search, thread pool executor
- 38 results per query per engine
- Redundancy: if one fails, the other covers

6 Query Categories:
1. AI tool launches and releases
2. AI apps, platforms, startups, SaaS
3. AI tool features, updates, integrations
4. AI directories and marketplaces
5. General new/just released/now available
6. Site filter: .com .io .ai .app .dev .tech .org

70+ Blocked Domains:
- Social: YouTube, Reddit, LinkedIn, Twitter/X, TikTok
- News: TechCrunch, The Verge, Wired, Forbes, Bloomberg
- Aggregators: FutureTools, G2, Capterra, AlternativeTo
- Code: GitHub, GitLab, Bitbucket
- Other: Wikipedia, StackOverflow, Medium, Substack

URL Processing:
- Strip query params and fragments
- Deduplicate via set
- Every query has "after:YYYY-MM-DD" for 7-day recency


====================================================
SLIDE 7 — SCRAPING & RESILIENCE
====================================================

Title:
Async Scraping & Resilience

Async Batch Fetching:
- Shared aiohttp session per batch
- Batch size = 4 (configurable)
- asyncio.gather for parallel fetching
- tqdm progress bar

Retry & Backoff:
- 3 retries per URL
- Exponential backoff: 2s → 4s → 8s
- HTTP 503: auto-retry
- HTTP 403/404: non-retryable, record to blacklist
- 30-second timeout per request

Ethical Scraping:
- robots.txt checked and cached per domain
- Realistic browser User-Agent
- 12 HTTP headers mimicking real browser
- Batch throttling for politeness

Persistent Blacklist:
- Tracks failures per domain
- 3 failures → domain auto-blacklisted
- Saved to data/blacklist.json after each run
- Loaded on startup — bad domains skipped automatically

Error Handling:
- AgentError base → ScrapingError, ExtractionError, ConfigError
- Every exception carries context (URL, step, status)
- All errors aggregated and reported at end

SLIDE 8 — LLM EXTRACTION
====================================================

Title:
LLM Extraction & Classification

Model: Azure OpenAI GPT-4o
- Temperature: 0.4
- Max tokens: 800
- API: AsyncAzureOpenAI

Prompt Design:
- System: "Expert AI tool analyst — JSON only, no hallucination"
- User: 3-step template with exact JSON schema
- Prompts in prompts/ directory — editable without code changes

Fields Extracted (per tool):
- Title
- Website
- Core Functionality (1-2 sentences)
- Target Audience
- Key Features (list)
- Pricing
- Source URL
- Tags
- Publish Date (ISO 8601)
- ai_tool_annotation: "ai_tool" or "not_ai_tool"

Why Single Call?
- 50% fewer API calls vs separate extract + classify
- Lower latency
- Classification benefits from extraction context

Throttling:
- Semaphore concurrency = 2
- Rate limit retry: 3 attempts, 10s delay

Robustness:
- HTML truncated to 15,000 chars
- Regex fallback extracts JSON from LLM output
- Strips markdown fences if present
- Fallback ToolInfo on parse failure

SLIDE 12 — DESIGN DECISIONS
====================================================

Title:
Key Design Decisions

1. Dual Search Engines
   Broader coverage + redundancy if one API fails

2. Single LLM Call
   Extract + classify in one call; 50% fewer API calls

3. 70+ Domain Blocklist
   Aggressive noise removal before scraping begins

4. Async Everything
   aiohttp + asyncio for massive I/O throughput

5. Externalized Prompts
   Edit prompts without code changes or redeployment

6. Persistent Blacklist
   3-strike rule; bad domains auto-skipped in future runs

7. Centralized Config
   One file (constants.py) to tune batch size, queries, headers

8. Custom Exceptions
   Error hierarchy with context dicts for debugging
